# Product Guidelines: Dual-Prediction Hybrid Text Classification System

## Voice & Tone
- **Professional & Technical**: All communication, including UI labels, API responses, and documentation, must be clear, precise, and data-driven. Avoid jargon where simple terms suffice, but maintain technical accuracy.

## Visual Identity (Branding)
- **Modern Clean (Shadcn)**: Leverage the default Shadcn UI aesthetic for a professional and contemporary feel.
- **Banking Professional (Blue/White)**: Incorporate a blue and white color palette to evoke trust and security, appropriate for a banking-focused tool.
- **High Contrast & Accessibility**: Ensure high contrast in all data visualizations and UI elements to maximize readability and accessibility.

## User Experience Principles
- **Speed & Efficiency**: Prioritize low-latency inference and immediate feedback. The transition from text input to classification results should feel instantaneous.
- **Clarity & Transparency**: Uncertainty warnings and classification results must be unmistakable. The user should always know when the model is confident and when it is not.
- **Simplicity & Focus**: Maintain a focused, single-page dashboard layout that minimizes distractions and guides the user through the 3-step instructional process.

## Documentation Standards
- **Comprehensive & Actionable**: Technical documentation must include both in-depth architectural details (e.g., hyperparameter grid search results, PyTorch model structures) and clear, actionable setup and usage guides.
- **Data-Driven Insights**: Use Matplotlib charts and Markdown tables to present model performance metrics and validation results clearly.
